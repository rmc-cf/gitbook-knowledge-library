# 架构风格



| 架构风格名称 | 数据处理方式                   | 系统拓展方式                   |
| ------ | ------------------------ | ------------------------ |
| 仓库     | 数据存储在中心仓库，处理流程独立，支持交互式处理 | 数据与处理解耦合，可动态添加和删除处理组件    |
| 管道过滤器  | 数据驱动机制，处理流程事先确定，交互性差     | 数据与处理紧密关联，调整处理流程需要系统重新启动 |
|        |                          |                          |
|        |                          |                          |
